Section 2
a) We might prefer to train on UTF-8 Encoded bytes because it uses less bytes to compress,
which means that there will be less to control. This only happens for ASCII characters though,
where UTF-8 only uses one byte. But, for non-ASCII characters it can use 3-4 bytes, whereas
UTF-16 tries to always use 2. UTF-32 has 4 bytes per character no matter what, and is better
when you need a fixed amount.
